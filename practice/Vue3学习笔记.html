🟢 Vue 3 学习笔记

⚖️ 一、Options API vs Composition API

Vue 3 提供了两种编写组件的方式，Composition API 是 Vue 3 的新特性，也是官方推荐的方式。

📋 Options API（选项式 API）

Vue 2 的传统写法，通过选项来组织代码。

<script lang="ts"> export default { // 组件名 name: 'Person', // 数据 data() { return { name: '张三', age: 18 } }, // 计算属性 computed: { fullInfo() { return this.name + ' - ' + this.age + '岁' } }, // 方法 methods: { changeName() { this.name = '李四' } }, // 生命周期 mounted() { console.log('组件挂载了') } } </script>
特点：选项分散，逻辑按选项类型组织（data、methods、computed 分开），适合简单组件和初学者。

🧩 Composition API（组合式 API）

Vue 3 新特性，通过函数来组织代码。

<script setup lang="ts"> import { ref, computed, onMounted } from 'vue' // 数据 const name = ref('张三') const age = ref(18) // 计算属性 const fullInfo = computed(() => { return name.value + ' - ' + age.value + '岁' }) // 方法 function changeName() { name.value = '李四' } // 生命周期 onMounted(() => { console.log('组件挂载了') }) </script>
特点：逻辑聚合，相关代码放在一起，适合复杂组件和大型项目，也是 Vue 3 官方推荐写法。
对比项Options APIComposition API
代码组织按选项类型分散（data、methods、computed）按功能逻辑聚合
this 指向需要使用 this 访问数据和方法不需要 this，直接使用变量
代码复用使用 mixins（容易命名冲突）使用组合函数（更清晰）
TypeScript 支持类型推断较弱类型推断更好
学习曲线较低，适合初学者稍高，但逻辑更清晰
适用场景简单组件、小型项目复杂组件、大型项目（推荐）

🔧 二、setup 概述

setup 是 Composition API 的入口函数，在组件创建之前执行，用于定义响应式数据、计算属性、方法、生命周期等。

2.1 setup 的执行时机

1
beforeCreate — 组件实例初始化
2
setup() — 在此执行！（在 beforeCreate 之后，created 之前）
3
created — 数据观测完成
4
beforeMount → mounted — DOM 挂载
setup 在组件的 beforeCreate 钩子之后、created 钩子之前执行。此时组件实例还未创建完成，所以 不能使用 this！

2.2 setup 的基本写法

<script lang="ts"> import { ref } from 'vue' export default { name: 'Person', /* setup 函数 */ setup() { // 定义数据 let name = '张三' // 普通变量，不是响应式的 let age = 18 // 普通变量，不是响应式的 // 定义方法 function showInfo() { alert(name + '今年' + age + '岁') } // 返回值：将数据和方法暴露给模板使用 return { name, age, showInfo } } } </script>

↩️ 三、setup 的返回值

setup 函数必须返回一个对象，对象中的属性和方法可以在模板中直接使用。

3.1 返回对象

setup() { const name = '张三' const age = 18 function sayHello() { console.log('Hello!') } // 返回对象，模板中可以直接使用 name、age、sayHello return { name, // 等价于 name: name age, // 等价于 age: age sayHello // 等价于 sayHello: sayHello } }

3.2 返回渲染函数（较少用）

import { h } from 'vue' setup() { return () => h('h1', 'Hello Vue 3') // 返回渲染函数 }
大多数情况下，setup 返回一个包含数据和方法的对象即可。返回的内容可以在模板中通过 {{ name }} 或 @click="sayHello" 直接使用。

🔗 四、setup 与 Options API 的关系

Vue 3 允许在同一个组件中同时使用 setup 和 Options API 的选项，但有一些规则和注意事项。

4.1 可以同时使用

<script lang="ts"> export default { name: 'Person', // Options API 的数据 data() { return { a: 100 } }, // setup 中定义的数据 setup() { let b = 200 return { b } } } </script> <template> <div> <p>a = {{ a }}</p> <!-- 来自 data()，可以访问 --> <p>b = {{ b }}</p> <!-- 来自 setup()，可以访问 --> </div> </template>

4.2 访问规则

场景能否访问说明
setup 中访问 data 的数据❌ 不能setup 执行时 data 还未初始化
setup 中访问 methods 的方法❌ 不能setup 执行时 methods 还未初始化
data 中访问 setup 的数据❌ 不能data 中不能使用 setup 返回的数据
methods 中访问 setup 的数据❌ 不能methods 中不能使用 setup 返回的数据
模板中访问 setup 和 data✅ 都能模板中可以同时使用两者返回的数据
生命周期钩子中访问 setup✅ 能生命周期在 setup 之后执行
setup 和 Options API 的数据是隔离的！ setup 中不能通过 this 访问 data 和 methods，Options API 中也不能访问 setup 的数据。建议一个组件中只使用一种风格，不要混用。

🍬 五、setup 语法糖（script setup）

Vue 3.2+ 引入了 <script setup> 语法糖，让 Composition API 的写法更简洁，是官方强烈推荐的写法。

5.1 传统 setup 写法 vs 语法糖写法

❌ 传统 setup 写法（较繁琐）

<script lang="ts"> import { ref } from 'vue' export default { name: 'Person', setup() { const name = ref('张三') function changeName() { name.value = '李四' } return { name, changeName } } } </script>

✅ setup 语法糖（推荐！）

<script setup lang="ts"> import { ref } from 'vue' // 直接写变量和方法，自动暴露给模板 const name = ref('张三') function changeName() { name.value = '李四' } // 不需要 return，自动暴露！ </script>

5.2 语法糖的优势

  1. 更少的样板代码：不需要 export default 和 return
  2. 自动暴露：定义的变量和方法自动暴露给模板，无需手动 return
  3. 更好的 TypeScript 支持：类型推断更精确
  4. 更好的 IDE 支持：代码提示和跳转更准确
  5. 组件名自动推断：文件名即组件名（也可通过 defineOptions 指定）

5.3 语法糖的注意事项

<script setup lang="ts"> // 1. 组件自动暴露，不需要 return const count = ref(0) // 2. 使用 defineProps 声明 props（宏函数，无需导入） const props = defineProps<{ title: string age?: number // ? 表示可选 }>() // 3. 使用 defineEmits 声明事件（宏函数，无需导入） const emit = defineEmits<{ update: [value: number] }>() // 4. 使用 defineExpose 暴露给父组件 defineExpose({ count, reset() { count.value = 0 } }) // 5. 指定组件名 defineOptions({ name: 'MyComponent' }) </script>
defineProps、defineEmits、defineExpose、defineOptions 是编译器宏，不需要 import 导入，只在 <script setup> 中可用。

📌 六、ref 创建基本类型的响应式数据

ref 是 Vue 3 中用于创建响应式数据的函数，适合定义基本类型（string、number、boolean 等）的响应式数据。

6.1 ref 的基本用法

<script setup lang="ts"> import { ref } from 'vue' // 定义响应式数据（基本类型） const name = ref('张三') // string 类型 const age = ref(18) // number 类型 const isStudent = ref(true) // boolean 类型 // 在 JavaScript/TypeScript 中修改值 function changeName() { name.value = '李四' // 必须通过 .value 修改！ } function addAge() { age.value += 1 // 必须通过 .value 修改！ } </script> <template> <div> <!-- 在模板中直接使用，不需要 .value --> <p>姓名：{{ name }}</p> <p>年龄：{{ age }}</p> <p>是否学生：{{ isStudent }}</p> <button @click="changeName">修改名字</button> <button @click="addAge">年龄+1</button> </div> </template>

6.2 ref 的核心要点

  1. 定义 ref(初始值) 创建一个响应式引用对象
  2. JS/TS 中读取 必须通过 name.value 访问值
  3. JS/TS 中修改 必须通过 name.value = 新值 修改
  4. 模板中读取 直接使用 {{ name }}，不需要 .value
  5. 重要 忘记 .value 是新手最常见的错误！
为什么需要 .value？
ref 返回的是一个 RefImpl 对象（引用实现对象），真正的值存储在对象的 .value 属性中。Vue 通过这种方式拦截对 .value 的访问和修改，从而实现响应式。模板中 Vue 会自动解包，所以不需要 .value。

🗂️ 七、reactive 创建对象类型的响应式数据

reactive 是 Vue 3 中用于创建对象类型（对象、数组、Map、Set 等）响应式数据的函数。

7.1 reactive 的基本用法

<script setup lang="ts"> import { reactive } from 'vue' // 定义对象类型的响应式数据 const person = reactive({ name: '张三', age: 18, hobbies: ['篮球', '音乐'] }) // 定义数组类型的响应式数据 const list = reactive([1, 2, 3]) // 修改对象属性（不需要 .value！） function changeName() { person.name = '李四' // 直接修改属性，不需要 .value } function addAge() { person.age ++ // 直接修改属性 } function addHobby() { person.hobbies.push('游泳') // 数组方法也支持 } function addItem() { list.push(list.length + 1) // 数组操作 } </script> <template> <div> <h3>对象类型数据</h3> <p>姓名：{{ person.name }}</p> <p>年龄：{{ person.age }}</p> <p>爱好：{{ person.hobbies }}</p> <h3>数组类型数据</h3> <p>列表：{{ list }}</p> <button @click="changeName">修改名字</button> <button @click="addAge">年龄+1</button> <button @click="addHobby">添加爱好</button> <button @click="addItem">添加列表项</button> </div> </template>

7.2 reactive 的核心要点

  1. 定义 reactive(对象) 创建对象类型的响应式数据
  2. 访问 直接通过 person.name 访问，不需要 .value
  3. 修改 直接通过 person.name = '新值' 修改
  4. 限制 只能用于对象类型（对象、数组、Map、Set），不能用于基本类型
  5. 注意 不能替换整个对象（会失去响应式），只能修改属性
reactive 的局限性：
1. 不能对 reactive 对象直接赋值新对象（会丢失响应式）
2. 解构 reactive 对象的属性会失去响应式
3. 这些情况下建议使用 ref 包裹对象

📦 八、ref 创建对象类型的响应式数据

ref 不仅可以定义基本类型，也可以定义对象类型的数据。当 ref 的参数是对象时，内部会自动通过 reactive 转换为响应式对象。

8.1 ref 定义对象类型

<script setup lang="ts"> import { ref } from 'vue' // ref 定义对象类型（内部自动使用 reactive 处理） const person = ref({ name: '张三', age: 18, hobbies: ['篮球', '音乐'] }) // ref 定义数组类型 const list = ref([1, 2, 3]) // 修改对象属性（需要 .value！） function changeName() { person.value.name = '李四' // 先 .value 再访问属性 } function addAge() { person.value.age ++ // 先 .value 再访问属性 } function addHobby() { person.value.hobbies.push('游泳') } function replacePerson() { // ref 可以替换整个对象（仍然保持响应式！） person.value = { name: '王五', age: 25, hobbies: ['编程'] } } </script> <template> <div> <p>姓名：{{ person.name }}</p> <!-- 模板自动解包，不需要 .value --> <p>年龄：{{ person.age }}</p> <p>爱好：{{ person.hobbies }}</p> <button @click="changeName">修改名字</button> <button @click="replacePerson">替换整个对象</button> </div> </template>

8.2 ref 对象 vs reactive 对象

操作ref 对象reactive 对象
修改属性person.value.name = '新值'person.name = '新值'
替换整个对象✅ person.value = {...} 保持响应式❌ person = {...} 丢失响应式
解构属性✅ 可用，保持响应式❌ 会失去响应式
作为参数传递✅ 传递的是 Ref 对象✅ 传递的是 Proxy 代理对象
当需要替换整个对象或解构属性时，优先使用 ref 包裹对象。如果只是修改对象属性且不需要替换整个对象，reactive 更简洁。

⚖️ 九、ref 和 reactive 对比总结

📌 ref

import { ref } from 'vue' // 基本类型 const name = ref('张三') const age = ref(18) // 对象类型 const person = ref({ name: '张三' }) // JS 中修改 name.value = '李四' person.value.name = '李四' // 模板中使用 // {{ name }} {{ person.name }}

🗂️ reactive

import { reactive } from 'vue' // 只能用于对象类型 const person = reactive({ name: '张三', age: 18 }) const list = reactive([1, 2, 3]) // JS 中修改 person.name = '李四' // 不需要 .value list.push(4) // 模板中使用 // {{ person.name }} {{ list }}
对比项refreactive
数据类型✅ 基本类型 + 对象类型❌ 只能对象类型（对象、数组、Map、Set）
JS/TS 中访问需要 .value不需要 .value，直接访问属性
模板中访问不需要 .value（自动解包）直接访问属性
替换整个对象✅ 可以，保持响应式❌ 会丢失响应式
解构✅ 可用，保持响应式❌ 会失去响应式
内部实现RefImpl 对象Proxy 代理
推荐场景基本类型、需要替换/解构的对象复杂对象、只修改属性不替换

🎯 选择建议

  • 基本类型 必须用 ref（string、number、boolean）
  • 对象类型 ref 和 reactive 都可以，但推荐统一使用 ref（避免混用导致困惑）
  • 需要替换 用 ref（如表单重置、切换数据）
  • 需要解构 用 ref（如从对象中解构出某个属性）
最佳实践：实际开发中建议统一使用 ref 来定义所有响应式数据，这样访问规则一致（都是 .value），减少心智负担。团队规范中通常推荐只使用 ref。

📝 十、完整示例：Person 组件

综合运用所学知识，写一个完整的 Person 组件，对比 Options API 和 Composition API 的写法。

Options API 写法

<template> <div class="person"> <h2>姓名：{{ name }}</h2> <h2>年龄：{{ age }}</h2> <button @click="changeName">修改名字</button> <button @click="addAge">年龄+1</button> </div> </template> <script lang="ts"> export default { name: 'Person', data() { return { name: '张三', age: 18 } }, methods: { changeName() { this.name = '李四' }, addAge() { this.age ++ } } } </script> <style scoped> .person { background-color: aqua; padding: 20px; border-radius: 10px; } </style>

Composition API 写法（推荐）

<template> <div class="person"> <h2>姓名：{{ name }}</h2> <h2>年龄：{{ age }}</h2> <button @click="changeName">修改名字</button> <button @click="addAge">年龄+1</button> </div> </template> <script setup lang="ts"> import { ref } from 'vue' // 定义响应式数据 const name = ref('张三') const age = ref(18) // 定义方法 function changeName() { name.value = '李四' } function addAge() { age.value ++ } </script> <style scoped> .person { background-color: aqua; padding: 20px; border-radius: 10px; } </style>
